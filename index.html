このデータで何がしたいかを決め、分解する

✅ このワークのゴール 「このデータで何を知りたいか」が1行で言えて、適材適所マップの全工程に担当が割り当たっている状態。

まず「何がしたいか」を決める

GASを作り始める前に、このデータをAIに渡して何をしたいのかを決めます。

なぜ最初に決めるのか——目的が決まって初めて、「要る列・要らない列」が決められるからです。
目的なしでマスキングを始めると、「とりあえず全部消す」で分析に使えなくなるか、「とりあえず全部残す」で安全になりません。
列を残すか消すかは、目的から逆算します。

今日の共通お題では、この目的で進めてください 「毎月のお客様の声から、商品や対応の課題を見つけて、報告する」
→ だから商品名・数量・自由記述（お客様の声）は残す／氏名・連絡先は目的に関係ないから消す。
この順番で考えます。

自分の業務に転用するときは、この目的決めをAIとの壁打ちでやるのがおすすめです。
データの中身を貼る必要はありません——列名と業務の説明だけで相談できます。

毎月の◯◯業務のデータ分析を仕組み化したいです。まず「このデータで何を知りたいか」の整理を手伝ってください。
・データの列：（列名だけ書く。中身の行は貼らない）
・報告する相手：◯◯

コピペで貼るのも、ファイルごと読み込ませるのもNGです。
AIに渡してよいのは、このあとGASでマスキングした後のmaskedシートのデータだけ。

そもそもGASとは？

GAS（Google Apps Script）＝Googleスプレッドシートに標準搭載されている自動化機能です。
追加インストール不要・無料で、スプレッドシートのメニューから実行できます。ExcelでいうVBAにあたります。

コードは書けなくてOK GASのコードを書くのはAIの仕事です。あなたがやるのは「何をしてほしいか」を伝えて、返ってきたコードを貼って実行するだけ。
今日はその頼み方の「型」から練習します。

依頼の5要素（今日いちばん大事な型）

「マスキングして」とだけ頼むと、列の扱いがズレます。
決めるのはこの5つだけです。

要素内容
A. ゴール個人情報を消して"最悪漏れても問題ない状態"にするGASを書いて／非エンジニアがコピペで動く完成形・日本語コメント付き
B. 入力どのシートに、どんな列があるか（列名を全部書き出す）
C. 列ごとの扱いそのまま使う／削除する／仮名化する（メール→ID化）／一部だけ残す（住所→都道府県）／自由記述は形式明確なものだけ機械置換
D. 出力どのシートに・どんな列で
E. 使い方メニューから実行できるように、等
禁止則（必ず守る） AIに渡すのは構造（列名）だけ。
中身の行は1行も貼らないでください。
「例を見せた方が伝わる」と思ってサンプル行を貼った瞬間、生データ送信になってしまいます。
例が要るときはダミー値を自作してください。

W3

分析スキルを、その場で作る

✅ このワークのゴール 分類カテゴリと報告形式が自分の納得で決まり、「このスキルでもう一度」と頼める自分のスキルが1つできている状態。

雛形は配りません

今日の分析スキルは、完成品を配って書き換えてもらう形ではありません。
マスク済みCSVをまずAIに直接分析させ、分類・集計・報告形式が納得いく形になるまで対話で固めてから、その手順をスキルにします。

「今の手順をスキルにして」と頼むだけで通ります——スキルとは「判断基準を固定して繰り返し使えるようにする仕組み」だと、雛形の書き換えではなく自分の判断の固定として体験してもらうためです。

STEP 1試行錯誤する雛形なし、対話でその場分析
→
STEP 2分類・報告の形が固まる納得いくまで調整
→
STEP 3スキルとして固定「今の手順をスキルにして」

作業⑤アドホック分析（雛形なしで、その場で試行錯誤する分析）→ スキル化

🖥 操作するもの：スプレッドシート（CSVダウンロード）→ Claude Code / Codex

  1. maskedシートを開く→「ファイル」→「ダウンロード」→「CSV」→今日の作業フォルダへ移動する（masked_2026-04.csvなど月がわかる名前にリネームしておく。作業⑦の複利デモでこのファイルをもう一度使います）
  2. Claude Code / Codexに、そのCSVを直接渡して分析を依頼する（分類カテゴリ・集計内容・報告形式を対話で調整する）
  3. 納得いく形になったら「今の手順をスキルにして」と依頼する
依頼する前に決めておくこと 何を目的にデータを分析させるのか、先に一文で決めておきましょう。
目的が曖昧なまま投げると、分類の切り口も報告の形もブレて、後の「スキル化」で毎回作り直しになります。
最初の分析依頼の例（自分の言葉に変えてOK）
このCSVは、個人情報をマスキングした注文データ（顧客アンケートのコメント付き）です。
お客様のコメントを内容ごとに分類して、月ごとに件数を集計してください。

返ってきた結果を見て、気になるところを対話で直します。
「この5件はどっちのカテゴリ？」「なぜこの分類にした？」——聞き方を変えると、返ってくるものが変わります。
ここは答えをもらう時間ではなく、自分の判断基準を言葉にする時間です。

スキル化の前に、決めておくこと

  • 分類カテゴリ：「今日はこのカテゴリで固定」と言い切れる状態まで決める。どのカテゴリにも収まらない新傾向は「別枠」で報告させる
  • 報告形式：毎月同じ見た目・同じ並びで出す形を決める（ものさしを固定するから、先月と今月を比べられる）
  • 要確認リスト：「自由記述に人名が含まれる行を一覧にして」と頼み、機械で消しきれなかった個人情報を人の目でチェックする。これが適材適所の「人」の仕事です
スキル化の依頼（そのまま送ってOK）
今の分析の手順（分類のやり方・集計の仕方・報告の形式）を、繰り返し使えるスキルにしてください。
次に新しい月のデータが来たときも、同じ基準で処理できるようにしたいです。
大事な理由 分類カテゴリをAI任せにすると、月ごとにラベルが揺れて月次比較が崩れます。
「納得したカテゴリをスキルに固定する」ことが、複利（=毎月同じ基準で比較できる）の土台になります。
Codexで進める方へ 「アドホック分析→納得いったらスキル化」という順番・依頼の言葉づかいはそのまま使えます。
うまく通らない場合は、講師の完成スキルの画面共有デモにフォールバックしてください。
W4

ダッシュボード化と、複利の確認

✅ このワークのゴール 3ヶ月分のダッシュボードが出て、6月の変化が読み取れている状態（コードやスキルの書き換えはゼロ）。
なぜ、分析とダッシュボード化を別のスキルにしておくのか 分類・集計は「毎月同じ基準で比べたい」ので、判断基準を固定してスキルにします。
ダッシュボード化も、慣れてくれば同じようにスキル化して構いません。
ただし1つのスキルにまとめてしまうと、「今月はこのグラフを目立たせたい」といった見せ方だけ変えたいときにも分析ロジックごと触ることになり、変更の影響範囲が混ざります。
「分類基準を固定するスキル」と「見せ方を差し替えられるスキル」を分けておくのが、適材適所です。

A3

claspで一歩先へ

今日のGASは、Googleのエディタ上でAIに書いてもらう形でした。
ここから先は、その一歩先——「こういう風にGASを使う世界もある」という紹介です。

claspとは

clasp（クラスプ）は、Googleが公式に出しているコマンドラインツールです。
GASのコードを、あなたのPCとGoogle側とで同期してくれます。
Google自身が配っているものなので、その点は安心して使えます。

今日は、AIに書いてもらったコードをコピーして、GASエディタに貼り付けました。
claspを入れると、この往復がなくなります。
Claude Code / Codexがコードを直接書き換え、コマンド1つでGoogle側に反映されます。

できることどういうことか
コピペの往復が消えるAIがGASのコードを直接書き換える。エディタに貼り付ける手作業がなくなる
コードが手元に残るファイルとしてPCに置ける＝AIがコード全体を読んで直せる。変更の履歴も残せる
反映も取り込みも1コマンドclasp pushで手元の変更をGoogleへ送る／clasp pullでGAS側の変更を手元に取り込む
当日は紹介のみ Node.js導入・clasp login・Apps Script API有効化など、環境ごとの分岐が増えるため、当日ハンズオンでは手を動かしません。
ここから先は持ち帰って、自分のペースで試してください。

事前準備（1回だけ）

  1. nodejs.orgからNode.jsをインストールする（ターミナルでnode -vが通ればOK）
  2. ターミナルでnpm install -g @google/claspを実行し、claspをインストールする
  3. clasp loginを実行し、ブラウザでGoogleアカウントにログインする

今日のスプレッドシートを、そのまま触る（clasp clone）

今日マスキングGASを書き込んだ、あのスプレッドシートの続きをやる場合はこちらです。
まず、スプレッドシート側のスクリプトIDを調べます。

  1. スプレッドシートの「拡張機能」→「Apps Script」でエディタを開く
  2. 左サイドバーの⚙️（プロジェクトの設定）を開く
  3. 「スクリプトID」をコピーする

Claude Code / Codexの作業フォルダの中で、次のコマンドを実行します。

既存プロジェクトを手元に落とす
clasp clone コピーしたスクリプトID

今日のGASのコードが、手元にファイルとして落ちてきます。
あとはClaude Code / Codexに「この機能を追加して」と頼み、clasp push --forceでスプレッドシート側に反映するだけです。

ゼロから新しく作る（clasp create）

新しいスプレッドシートごと作る場合は、作業フォルダの中で次を実行します。

新規プロジェクト作成
clasp create --type sheets --title "プロジェクト名"
clasp push --force

これでスプレッドシートに紐づいたGASプロジェクトができます。
以降はClaude Code / Codexに「この機能を追加して」と頼むだけでローカルのコードが書き変わり、clasp push --forceでクラウドに反映されます。

⚠️ ここでほぼ全員が引っかかる関門 実際に業務ツールをclasp化したときに、順番に全部踏みました。先に知っておけば数分で通過できます。

関門1：トークン期限切れ

invalid_grant / invalid_rapt というエラーが出たら、認証が切れています。clasp loginで再認証してください。

関門2：Apps Script APIが無効

User has not enabled the Apps Script APIというエラーが出たら、Apps Scriptの設定ページでトグルをONにします（反映まで数分かかることがあります）。
一覧表示のような読み取り系コマンドは通っていても、作成系コマンドで初めて出るエラーです。

関門3：clasp createがマニフェストを上書きする（★最重要）

clasp createで新規作成すると、権限設定（oauthScopes）が空の初期状態でマニフェストが上書きされます。
すでに動いていたGASを複製したつもりでも外部APIへのアクセス権限が消えてしまい、403 ACCESS_TOKEN_SCOPE_INSUFFICIENTのようなエラーで詰まります。

対策 create前に元のappsscript.jsonをバックアップしておき、作成後にoauthScopesの中身を復元してpushし直してください。

関門4：scopeを足しただけでは反映されない

マニフェストに権限（scope）を追加してpushしても、それだけでは反映されません。
GASエディタを再読み込みし、関数を一度手動実行して再承認する必要があります。

関門5：GASがGoogle管理のGCPにぶら下がる（★）

clasp createで作ると、Googleが管理する共有GCPプロジェクトの配下に置かれ、そこはAPIの有効化などの操作ができません。
外部APIを使おうとすると403 SERVICE_DISABLEDのようなエラーになります。

対策 GASエディタの「プロジェクトの設定」→「GCPプロジェクトを変更」から、自分が管理者になっているGCPプロジェクト番号に付け替え、そのプロジェクト側でAPIを有効化します。

関門6：機能を足すたびにscopeが増える

トリガー操作やGmail送信など、新しい機能を使うたびに必要なscopeが増えます。
「指定された権限では〜を呼び出すことができません」というエラーが出たら、該当のscopeをマニフェストに追加→push→GASエディタ再読込→関数を手動実行して再承認、の順で直します。

Tips：ファイルが黙ってpushされないとき .claspignoreを許可リスト形式（**/** + !ファイル名）で使っていると、後から追加したファイルが対象外のままpushされないことがあります。
clasp pushは成功したように表示されるので気づきにくいです。clasp statusのTracked/Untrackedで確認するか、一時フォルダにcloneして実際に反映されているか確認するのが確実です。

Web制作の現場でも使われるサービスです。

GitHubは、世界中の開発者がコードを置いている場所です。

Troubleshooting

よくある質問

はじめる前の不安と、当日よくある詰まりを集めました。
質問を開くと答えが出ます。

Q1スプレッドシートに個人情報を入れて大丈夫なんですか？
今日配るデータはダミーです。
実在の個人情報は入っていないので、そのまま進めて大丈夫です。

そのうえで、この不安には大事な区別が隠れています。「スプレッドシートに置く」と「AIに渡す」は別の話です。
スプレッドシートはあなたのGoogleアカウントの中にあります。
GASもGoogleの中だけで動くので、生データが外部のAIに送られることはありません。
